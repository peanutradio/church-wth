# A. 디자인 — "있어 보이게"

## A-1. Hero(첫 화면) 리디자인 ⭐가장 큰 임팩트

**현재**: 전체 화면을 차지하는데 **그라데이션 배경 + 텍스트만** 있어 허전합니다. 버튼(CTA)도, 시선을 끄는 비주얼도 없습니다.

```
[ 현재 ]
┌──────────────────────────────┐
│   (연분홍 → 연보라 그라데이션)      │
│                              │
│     WE, THE CHURCH에           │
│     오신 것을 환영합니다           │
│     신앙과 일상이 함께 자라는 공간   │
│                              │
└──────────────────────────────┘
```

**개선안**:
```
[ 개선 ]
┌──────────────────────────────┐
│ [교회 공간/예배 사진 — 잔잔한 줌인]  │
│   (가독성용 부드러운 오버레이)      │
│                              │
│     WE, THE CHURCH            │
│     신앙과 일상이 함께 자라는 공간   │
│                              │
│   [ 예배 안내 ]  [ 온라인 예배 ]   │ ← 버튼(CTA)
│            ↓ (스크롤 유도)        │
└──────────────────────────────┘
```

- **배경**: 실제 교회 사진(예배/공간) 위에 부드러운 오버레이. 사진이 없으면 → 고급 그라데이션 + 은은한 패턴/움직임으로 임시.
- **CTA 버튼 2개**: `예배 안내 보기`(아래로 스크롤), `온라인 예배`(유튜브) — 방문자가 바로 행동하게.
- **스크롤 인디케이터**: 아래쪽 ↓ 깜빡임 — "더 있어요" 신호.
- **텍스트 위계**: 영문 타이틀은 크고 강하게, 한글은 우아하게(명조체 유지).

**효과**: 첫인상이 "정보 페이지" → "잘 만든 교회"로 확 바뀝니다.

---

## A-2. 스크롤 등장 애니메이션

**현재**: 코드에 `animate-fade-in-up` 클래스를 쓰고 있는데 **정의(키프레임)가 빠져 있어 실제로 작동하지 않습니다.** (Hero, About에 적용했지만 무반응 — 사실상 버그)

**개선안**:
- `index.css`에 `fade-in-up` 키프레임 추가 (지금 빠진 것 보충)
- 스크롤해서 섹션이 화면에 들어올 때 **스르륵 떠오르는** 효과 (IntersectionObserver)
- 카드들은 **순차적으로**(0.1초 간격) 등장 → 고급스러운 리듬

**효과**: 페이지를 내릴 때마다 콘텐츠가 살아 움직여 "정성 들인 사이트" 느낌.

---

## A-3. 실제 교회 사진 적용 📸 (사진 준비 필요)

**현재**: 일러스트 이미지(파스텔 성경 그림)만 사용 → 실제 교회의 모습이 안 보임.

**개선안** — 아래 위치에 실제 사진을 넣습니다:
- Hero 배경 (예배 전경 또는 공간)
- About 섹션 (공동체/모임 사진)
- (신설) 갤러리 섹션 — 교회 사진 3~6장 그리드

**필요한 사진** (있는 것만이라도):
- 예배 드리는 모습 / 강단
- 교회 공간(입구, 내부)
- 공동체(소그룹, 식사, 행사)
- 목회자/스태프

> 📌 휴대폰으로 찍은 것도 괜찮아요. 밝고 사람들이 자연스러운 사진이면 충분합니다.

**효과**: 일러스트보다 **신뢰감·생동감**이 훨씬 큽니다.

## A-4. 디테일 다듬기

- **헌금 섹션**: "계좌번호" 글자색이 어두운 그라데이션 배경에서 잘 안 보임 → 대비 조정
- **여백·정렬 리듬** 통일 (섹션 간 간격)
- **버튼/카드 그림자·둥근모서리** 시스템 정리 (지금도 좋지만 약간씩 제각각)

# B. 기능 — 참여와 편의

## B-1. 다음 예배 카운트다운 / 유튜브 라이브 자동 표시

**아이디어**:
```
┌──────────────────────────────┐
│   다음 주일예배까지              │
│      2일 14시간 30분            │
│   (또는: 🔴 지금 라이브 중!)     │
└──────────────────────────────┘
```
- 예배 시간표(주일 13:30 등)를 기준으로 **다음 예배까지 남은 시간** 표시
- 예배 시간이 되어 유튜브 라이브가 켜지면 → **자동으로 라이브 영상** 노출
- 위치: Hero 아래 띠 배너 또는 예배 안내 섹션 상단

**효과**: "이번 주 언제 모이지?" 를 한눈에. 온라인 참여 유도.

## B-2. 카카오맵으로 전환

**현재**: 구글 지도 사용 (한국에서는 길찾기·대중교통이 약함)

**개선안**: 카카오맵으로 교체 + **"길찾기" 버튼**(카카오내비/지도 앱 바로 연결). 한국 성도들에게 훨씬 친숙하고 편리.

> 필요: 카카오 지도 API 키(무료 발급) — 제가 발급 화면 안내해 드릴게요.

## B-3. 새가족 등록 / 기도제목 (온라인 폼)

- **새가족 등록**: 처음 오신 분이 간단히 인사·연락처 남기기 → 관리자 페이지에서 확인
- **기도제목**: 성도들이 기도 요청을 남기고(공개/비공개 선택) 함께 기도
- 이미 만들어둔 **연말정산 신청 폼 구조를 재활용**하면 빠르게 가능 (관리자 조회 탭도 동일 패턴)

## B-4. 교회 일정/행사 캘린더

- 절기·행사·특별예배 일정을 달력으로 안내
- 방법 ①: 구글 캘린더 임베드(가장 간단) / 방법 ②: 직접 입력형(관리자 페이지에서 등록)

# C. 기술 품질 (눈에 안 보이지만 중요)

## C-1. SEO — 페이지별 메타태그
**현재**: 모든 페이지가 `index.html`의 **고정 제목/설명**을 공유 → 검색엔진에서 페이지 구분이 약함.
**개선**: 페이지별(말씀/소식/연말정산 등) 제목·설명을 따로 지정 → 검색 노출↑, 카톡 공유 시 정확한 정보 표시.

## C-2. 로딩 속도
**현재**: 자바스크립트 한 덩어리가 **750KB**(엑셀 라이브러리 등 포함) → 첫 로딩이 무거움.
**개선**: 관리자 전용 코드(엑셀 등)를 **분리 로딩**, 이미지 **webp 변환** → 첫 화면이 빨라짐.

## C-3. 자잘한 정리
- ✅ 스크롤 애니메이션 버그(A-2에서 함께 해결)
- ⚠️ **Footer 약관에 `"사이트명"` 같은 미완성 placeholder 텍스트**가 남아있음 → 실제 교회명으로 교체
- 📷 카톡/페이스북 공유 미리보기 이미지(OG) → 로고 대신 보기 좋은 대표 이미지로
